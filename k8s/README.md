# Deploy on Minikube

## 1) Start Minikube and enable ingress

```powershell
minikube start
minikube addons enable ingress
```

## 2) Build images inside Minikube Docker

In PowerShell:

```powershell
minikube -p minikube docker-env | Invoke-Expression
docker build -t deploydude/api-server:latest ./api-server
docker build -t deploydude/reverse-proxy:latest ./s3-reverse-proxy
```

## 3) Create namespace

```powershell
kubectl apply -f ./k8s/namespace.yaml
```

## 4) Create env resources from your existing .env files

This keeps secrets out of git and passes env in the correct Kubernetes way.

```powershell
$MINIKUBE_IP = minikube ip

kubectl create secret generic api-server-env `
  --namespace deploydude `
  --from-env-file=./api-server/.env `
  --from-literal=PUBLIC_PROTOCOL=http `
  --from-literal=PUBLIC_BASE_DOMAIN=$MINIKUBE_IP.nip.io `
  --from-literal=PUBLIC_PORT=80 `
  --dry-run=client -o yaml | kubectl apply -f -

kubectl create configmap reverse-proxy-env `
  --namespace deploydude `
  --from-env-file=./s3-reverse-proxy/.env `
  --dry-run=client -o yaml | kubectl apply -f -
```

## 5) Deploy workloads and ingress

```powershell
kubectl apply -f ./k8s/api-server-deployment.yaml
kubectl apply -f ./k8s/api-server-service.yaml
kubectl apply -f ./k8s/reverse-proxy-deployment.yaml
kubectl apply -f ./k8s/reverse-proxy-service.yaml
```

Set the ingress hosts to your current Minikube IP (avoid hardcoded stale IPs):

```powershell
$IP = minikube ip
(Get-Content ./k8s/ingress.yaml) `
  -replace '192\.168\.49\.2', $IP `
  | kubectl apply -f -
```

## 6) Verify

```powershell
kubectl get pods -n deploydude
kubectl get svc -n deploydude
kubectl get ingress -n deploydude
```

## 7) Access

- Reverse proxy root (default folder):

```powershell
$IP = minikube ip
curl http://$IP.nip.io/
```

- API endpoint for Postman:

```powershell
$IP = minikube ip
curl -X POST "http://api.$IP.nip.io/project" `
  -H "Content-Type: application/json" `
  -d '{"gitURL":"https://github.com/<owner>/<repo>.git"}'
```

Postman URL:

```text
http://api.<minikube-ip>.nip.io/project
```

- Socket endpoint (no localhost):

```text
ws://socket.<minikube-ip>.nip.io
```

Subscribe channel format:

```text
logs:<projectSlug>
```

- Project URL generated by api-server will now be:

```text
http://<projectSlug>.<minikube-ip>.nip.io
```

No `.localhost` is used anymore.

## Troubleshooting

- Restart deployment after env updates:

```powershell
kubectl rollout restart deploy/api-server -n deploydude
kubectl rollout restart deploy/reverse-proxy -n deploydude
```

- Tail logs:

```powershell
kubectl logs -n deploydude deploy/api-server -f
kubectl logs -n deploydude deploy/reverse-proxy -f
```
